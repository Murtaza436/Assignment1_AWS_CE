AWS Architecture

Designed following AWS Well-Architected Framework

🔐

IAM

Least-privilege roles for EC2 instances. Separate policies for S3 read/write, CloudWatch logging, and API access.

🌐

VPC

Custom VPC with public subnets (ALB) and private subnets (EC2). NAT Gateway for outbound API calls. Two AZs for HA.

🖥

EC2 (Auto Scaling)

t3.micro instances in private subnets. Auto Scaling Group across AZ-A and AZ-B. Health checks replace failed instances.

🪣

S3

Two prefixes: media/ for event posters and cache/ for API response JSON. Versioning enabled.

⚖️

Elastic Load Balancer

Application Load Balancer in public subnets. Routes HTTPS to healthy EC2 targets. Sticky sessions disabled for stateless scaling.

🔗

Ticketmaster API

Periodic fetch via cron (every 15 min). Events cached to S3 so the app stays up even if the external API is down.
